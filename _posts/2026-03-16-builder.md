# Summary

- Discover Jenkins application v2.441; vulnerable to LFI from CVE-2024-23897
- python poc from godylockz; read users.xml to find users’s home dir in jenkins
- read config.xml to find hashed password; cracked with hashcat
- logged in to jenkins; open script console; reverse shell onto docker instance
- find credentials.xml; has encrypted root private key
- download hudson.util.Secret, master.key, credentials.xml and decrypt with pwn_jenkins jenkins_offline_decrypt.py tool to retrieve root private key
- ssh with root private key as root

There is a CVE-2024-23897.

Root Cause: Jenkins has a built-in CLI (accessible via HTTP or TCP port 50000). The CLI uses the args4j library for argument parsing.

args4j has a feature: if an argument starts with @, it reads that file and uses its contents as arguments:

```bash
command @/etc/passwd

↑

args4j reads this file and expands its contents inline
```

Jenkins did not disable this feature. The expansion happens before authentication/authorization checks on several commands.
